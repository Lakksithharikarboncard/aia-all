01 Problem Statement

AIA helps Indian businesses process financial documents across Accounts Payable, Accounts Receivable, Bank Statements, Dashboard insights, and GST Reconciliation. Users like the product, but adoption is blocked because users cannot sign up by themselves, cannot see pricing, cannot buy a plan, and cannot start processing without manual setup. Internal teams must handle onboarding, billing, and account setup manually.

02 High-Level Approach

Build a self-serve commercial flow around the company, not the individual user:

Signup → Onboarding → Pricing → Checkout or Configure → Payment activation → Connect Tally / Zoho → Create or fetch company → Dashboard setup → Usage tracking → Billing management

Product Principles

PrincipleDecision
Company owns billingUsage, credits, revenue, invoices, and subscription attach to organization
Paid plans unlock AIA CorePaid users get AP, AR, Statements, Dashboard, and Users & Access
Trial is preview-onlyFree users can explore but cannot process real documents
Billing must feel fairCharge only for successful processing
Credits control usagePlan credits reset monthly; top-up credits roll over
Webhooks are source of truthPayment provider webhook activates subscription and grants credits
Admin actions must be auditableEvery credit, billing, and webhook action must have an event trail

04 Credit and Usage Model

A credit is the internal unit used to track successful AI processing. User-facing copy should say documents and bank statements where possible. The billing page may show credits for transparency.

Credit Consumption

ActionCredit Cost
Bill / purchase invoice extraction1 credit per successfully processed document
Sales invoice extraction1 credit per successfully processed document
Bank statement parsing10 credits per statement in V1
Dashboard viewing0 credits
Failed processing0 credits
Duplicate upload detected before processing0 credits
GST reconciliationAdd-on access; not metered by credits in V1
Note. Use the simplified rule for V1: 1 bank statement = 10 credits. Do not charge separately by transaction count. Track transaction count internally for future pricing and analytics.

Consumption Order

  1. Use plan credits first.
  2. Use top-up credits second.
  3. Block billable processing if both are insufficient.

Reset Behavior

Credit TypeReset / Expiry
Plan creditsRefresh monthly on billing renewal
Used creditsReset monthly
Top-up creditsDo not expire
Trial creditsNot applicable in V1 — trial has no processing credits

07 User Stories

Signup and Company

  • As a user, I want to sign up with email, so I can create an account.
  • As a user, I want to sign up with phone, so I can create an account.
  • As a user, I want to verify with OTP, so my account stays secure.
  • As a user, I want a company created during signup, so my billing and usage have one owner.
  • As an admin, I want usage tied to company, so I can support multi-user accounts.

Onboarding

  • As a user, I want to answer optional business questions, so AIA can guide my plan choice.
  • As a user, I want to skip onboarding, so I can see pricing immediately.
  • As a user, I want my onboarding answers saved, so I do not repeat them.
  • As a Growth user, I want onboarding data in HubSpot, so I can qualify leads faster.

Pricing

  • As a user, I want to compare Free, Starter, Growth, and Custom, so I can choose the right path.
  • As a user, I want to see documents and statements included, so I can understand plan limits.
  • As a user, I want to see GST Recon as an add-on, so I can choose it only if needed.
  • As a user, I want to see quarterly and annual discounts, so I can save on longer billing periods.

Checkout

  • As a user, I want to pay for Starter, so I can start processing documents.
  • As a user, I want to pay for Growth, so I can process higher volume.
  • As a user, I want to pay for a custom package, so I can match pricing to my workload.
  • As a user, I want GSTIN to be optional, so I can complete checkout even if I do not have it handy.

Credits and Processing

  • As a user, I want credits deducted only after success, so failed jobs do not cost me.
  • As a user, I want duplicate uploads to cost 0, so I am not charged for accidental repeats.
  • As a user, I want partial batch billing, so I only pay for successfully processed items.
  • As a user, I want low-credit warnings, so I can avoid interruption.
  • As a user, I want to buy top-ups, so I can continue work without changing plans.

Admin and Support

  • As an admin, I want to view customer billing state, so I can resolve support issues.
  • As an admin, I want immutable credit logs, so I can audit disputes.
  • As an admin, I want to manually adjust credits with a reason, so I can correct issues.
  • As Sales, I want CPQ leads in HubSpot, so I can follow up with full context.

10 Acceptance Criteria

Signup

  • User can sign up with phone.

  • System creates billing customer after company creation.

Trial
